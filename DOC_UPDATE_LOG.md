# 喵选（Meowpick）前端更新日志 (Update Log)

本文档记录了项目自**初始版本**以来的核心更新与修改点。本次更新的主要里程碑是引入了**“建课提议（Proposal）”**系统，并围绕该系统进行了全面的 UI 升级与交互重构。

## 1. 核心功能新增：提议系统 (Proposal System)
在初始版本仅支持“课程吐槽（Comment）”的基础上，全面新增了“提议”功能。
*   **数据结构扩展**：在全局列表中引入了 `type: 'proposal'` 与 `type: 'comment'` 的区分。提议卡片专门增加了 `voteCount`（附议/投票数）和 `reason`（提议理由）等独立字段。
*   **提议详情路由**：新增了专门的提议详情页（`/pages/course/proposal-detail/index`），与传统的课程主页（`/pages/course/index/index`）相互独立。

## 2. 页面级重构与更新

### 2.1 搜索与选课页 (`choose-course/index.vue`)
*   **混合列表展示**：搜索结果现在可以同时展示“已收录课程”和“待建课提议”。
*   **UI 视觉重构**：
    *   引入现代化卡片设计（`24rpx` 圆角，轻量级阴影）。
    *   **提议卡片高亮**：为提议卡片加入了主题色渐变背景（`linear-gradient(135deg, #fff5f6 0%, #ffffff 100%)`）以及品牌红色的边框。
    *   去除了初期过于厚重的边缘指示条，改用色彩背景区分，使得界面“有区分度但不分轻重”。

### 2.2 “我的”发布页面 (`profile-view/index.vue`)
*   **全新结构**：废弃了初始版本简陋的列表，重新设计了具备大标题（“我的发布”）和统计数量的现代化 Header。
*   **胶囊筛选栏 (Sticky Bar)**：
    *   新增 “全部”、“吐槽”、“提议” 三个分类 Tab。
    *   赋予该菜单 **吸顶（Sticky）** 功能与毛玻璃（`backdrop-filter: blur`）效果。
    *   **深度体验优化**：增加了向上延伸 `100vh` 的隐形背景遮罩层（`z-index: -1`），彻底解决了页面下滑时列表内容从刘海屏或状态栏“穿透露出”的 Bug，保证了沉浸式的滚动体验。
*   **卡片操作交互**：
    *   卡片增加点击缩放反馈（`&:active { transform: scale(0.98); }`）。
    *   根据不同卡片类型定制了独立的长按菜单：针对“提议”特别引入了“修改（清空票数）”等防错机制提示。

## 3. 性能与代码健壮性优化
*   **样式抽离与复用**：两处核心列表（搜索页与我的页）中对应的 DOM 结构和 SCSS 被高度统一，降低了后续维护成本。
*   **路由解耦**：彻底剥离了粗暴的统一页面跳转，重写为基于卡片类型动态分发的路由（`onItemClick` 策略模式映射）。
